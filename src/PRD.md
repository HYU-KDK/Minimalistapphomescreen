# PRD: 오마이쿡 (Oh My Cook)
## LLM 기반 보유 재료 맞춤형 요리 추천 서비스

---

## 1. 제품 개요

**오마이쿡**은 자취생과 가정 주부를 위한 AI 기반 요리 추천 웹 서비스입니다. 사용자가 현재 보유한 재료와 개인 특성(알레르기, 요리 수준, 주방 환경 등)을 입력하면, LLM이 최적의 레시피를 실시간으로 추천하고 대화형으로 조리 과정을 안내합니다.

### 핵심 가치 제안
- **재료 낭비 최소화**: 냉장고 속 재료로 즉시 만들 수 있는 요리 제안
- **의사결정 피로 해소**: 무엇을 먹을지 고민하는 시간 단축
- **개인 맞춤화**: 알레르기, 조리 시간, 난이도 등을 고려한 추천
- **실용성 우선**: 대체 재료 제안으로 부족한 재료 문제 해결

---

## 2. 제품 비전

### 해결하려는 문제
현대의 자취생과 가정 주부는 다음과 같은 어려움을 겪습니다:

1. **선택 과부하**: 수많은 레시피 중 무엇을 만들지 결정하기 어려움
2. **재료 미스매치**: 레시피에 필요한 재료가 없어 포기하는 경우 빈번
3. **개인화 부족**: 기존 레시피는 개인의 알레르기, 주방 환경, 기술 수준을 고려하지 않음
4. **정보 분산**: 재료 관리, 레시피 검색, 조리법 상담이 각각 다른 플랫폼에 흩어져 있음

### 목적
오마이쿡은 **AI가 사용자의 상황을 이해하고 최적의 요리를 제안**함으로써, 요리 결정 과정을 단순하고 즐겁게 만듭니다. 사용자는 재료 고민 없이 건강하고 경제적인 식사를 준비할 수 있습니다.

- **대체 재료 자동 제안**: 부족한 재료를 유사 재료로 치환해주는 똑똑한 추천

## 4. 핵심 기능 및 상세 사양

### 4.1 사용자 인증
**기능**: 로그인 / 회원가입  
**가치**: 개인화된 재료 목록과 선호도를 안전하게 저장하고, 다양한 디바이스에서 연속적인 경험 제공

- 이메일 기반 회원가입
- 소셜 로그인 (Google, Kakao 등) 옵션
- 비밀번호 찾기 및 재설정

---

### 4.2 개인화 프로필 설정
**기능**: 사용자 특성 조사 및 저장  
**가치**: AI가 사용자 상황에 맞는 레시피만 추천하여 불필요한 선택지 제거

#### 수집 정보
- **요리 수준**: 초급 / 중급 / 고급
- **알레르기 정보**: 계란, 우유, 견과류, 갑각류 등
- **보유 주방 도구**: 전자레인지, 인덕션, 에어프라이어, 오븐 등
- **조리 가능 시간**: 10분 이하 / 10-30분 / 30분 이상
- **맵기 선호도**: 1-5단계
- **음식 스타일 선호**: 한식, 양식, 중식, 일식, 퓨전
- **주간 예산**: 사용자 입력 (선택)

#### UX
- 최초 가입 시 온보딩 플로우에서 단계별로 질문
- 이후 프로필 페이지에서 언제든 수정 가능

---

### 4.3 재료 관리
**기능**: 보유 재료 등록 및 관리  
**가치**: 레시피 추천의 기준이 되는 정확한 재료 목록 확보

#### 재료 등록 방법
1. **수동 입력**: 검색 또는 카테고리별 선택
2. **영수증 인식**: 사진 업로드 → OCR 처리 → 재료 자동 추가
3. **냉장고 사진 업로드**: 이미지 인식으로 재료 추출 (선택 기능)

#### 재료 정보
- 재료명
- 수량 (선택)
- 유통기한 (선택)
- 카테고리 (채소, 육류, 해산물, 유제품, 조미료 등)

#### 재료 목록 UI
- 카테고리별 그룹화
- 유통기한 임박 재료 알림 (선택 기능)
- 빠른 추가/삭제 인터페이스

---

### 4.4 AI 요리 추천 시스템
**기능**: 보유 재료와 개인 특성 기반 레시피 생성  
**가치**: 사용자가 고민하지 않아도 지금 당장 만들 수 있는 최적의 요리 제시

#### 추천 알고리즘 입력
- 사용자 보유 재료
- 개인화 프로필 (알레르기, 난이도, 시간, 맵기 등)
- 과거 선호 레시피 히스토리 (선택)
- 현재 인기 트렌드 (선택)

#### 추천 결과 정보
각 레시피 카드에 다음 정보 표시:
- **레시피명**
- **대표 이미지** (AI 생성 또는 Unsplash)
- **인분**: 1인분 / 2인분 / 4인분 등
- **예상 조리시간**: 분 단위
- **난이도**: ★☆☆ (쉬움) / ★★☆ (보통) / ★★★ (어려움)
- **맵기**: 🌶️ 1-5단계
- **음식 스타일**: 한식 / 양식 / 중식 / 일식 / 기타
- **칼로리**: 대략적인 칼로리 (kcal)
- **예상 비용**: 추가 구매 재료 기준
- **부족한 재료**: 표시 + 대체 재료 제안
- **사용 주방 도구**: 아이콘으로 표시

#### 대체 재료 제안
예시:
- 대파 → 냉동 파 / 건조 파
- 올리브오일 → 식용유 + 참기름
- 생크림 → 우유 + 버터

#### UX
- 홈 화면에서 "오늘의 추천" 3-5개 자동 표시
- 레시피 카드 클릭 시 상세 조리법 페이지 이동

---

### 4.5 상세 레시피 페이지
**기능**: 단계별 조리법 및 재료 상세 안내  
**가치**: 요리 초보자도 따라 할 수 있는 명확한 가이드 제공

#### 구성 요소
1. **재료 목록**
   - 보유 재료는 ✅ 체크 표시
   - 부족한 재료는 ❌ 표시 + 대체 재료 제안

2. **조리 단계**
   - 번호별 단계 (1. 재료 손질 → 2. 조리 → 3. 완성)
   - 각 단계마다 설명 텍스트 + 팁

3. **타이머 기능**
   - 단계별 필요 시 타이머 버튼 제공 (예: "3분간 볶기")

4. **AI 어시스턴트 버튼**
   - "AI에게 물어보기" 버튼 → LLM 대화창 열림

---

### 4.6 LLM 기반 대화형 어시스턴트
**기능**: 요리 중 질문 및 조언 제공  
**가치**: 실시간으로 궁금증을 해결하고, 상황에 맞는 조리법 조정 가능

#### 사용 예시
- "불 세기를 어떻게 조절하나요?"
- "간장 대신 된장 써도 되나요?"
- "고기가 익었는지 어떻게 알 수 있나요?"
- "2인분으로 늘리려면 재료를 얼마나 추가하나요?"

#### 기술 구현
- OpenAI GPT-4 또는 Google Gemini API 활용
- 현재 레시피 컨텍스트를 프롬프트에 포함
- 대화 히스토리 유지 (세션 기반)

#### UX
- 채팅 인터페이스
- 음성 입력 옵션 (선택 기능)
- 이전 대화 기록 저장

---

### 4.7 인기 요리 트렌드
**기능**: 현재 다른 사용자들이 많이 만드는 요리 표시  
**가치**: 계절, 유행에 맞는 레시피 발견 및 영감 제공

#### 표시 정보
- 주간 TOP 10 인기 레시피
- 조회수 / 완성 횟수 기준 정렬
- 계절별 추천 (예: 여름 - 냉국, 겨울 - 찌개)

#### UX
- 홈 화면 하단 또는 별도 탭
- 카드형 레이아웃
- 클릭 시 레시피 상세 페이지 이동

## 5. 제안하는 추가 기능

### 5.1 요리 완성 기록 및 평가
**목적**: 사용자 만족도 피드백 수집 및 추천 알고리즘 개선

- 요리 완성 후 "완료" 버튼
- 별점 (1-5) 및 간단한 후기 입력
- 완성 사진 업로드 (SNS 공유 옵션)
- 나만의 레시피 북에 저장

### 5.2 주간 식단 플래너
**목적**: 장기적인 식사 계획 및 재료 구매 최적화

- 일주일치 식단 자동 생성
- 요일별 레시피 배정
- 필요한 재료 총 목록 자동 생성 (장보기 리스트)
- 예산 내 식단 구성

### 5.3 영양 정보 대시보드
**목적**: 건강 관리 및 균형 잡힌 식단 지원

- 하루/주간 섭취 칼로리 그래프
- 영양소 균형 (탄수화물, 단백질, 지방, 비타민)
- 알레르기 재료 자동 회피 확인

### 5.4 소셜 기능
**목적**: 커뮤니티 형성 및 사용자 참여 증대

- 레시피 공유 (친구에게 추천)
- 완성 사진 갤러리
- 사용자 리뷰 및 팁 공유

### 5.5 음성 안내 모드
**목적**: 요리 중 핸즈프리 조작 지원

- 조리 단계 음성 읽기
- "다음 단계" 음성 명령
- 타이머 음성 설정

### 5.6 재료 유통기한 알림
**목적**: 식재료 낭비 최소화

- 유통기한 3일 전 푸시 알림
- 해당 재료로 만들 수 있는 요리 즉시 추천

## 6. 타겟 사용자 페르소나

### 페르소나 1: 대학생 자취러 "민지" (22세, 여성)
- **직업**: 대학생 (경영학과 3학년)
- **거주**: 서울 원룸 자취
- **요리 수준**: 초급 (라면, 계란후라이 가능)
- **주방 환경**: 1구 인덕션, 전자레인지, 냉장고
- **목표**: 배달비 절약, 건강한 식사, 빠른 조리 (10-15분)
- **페인 포인트**: 
  - 재료 사도 다 쓰기 전에 상함
  - 레시피 보면 없는 재료가 항상 있음
  - 무엇을 만들지 결정하는 게 귀찮음
- **행동 패턴**: 
  - 저녁 9시 이후 귀가 후 요리
  - 주 1회 마트 장보기
  - 인스타그램에서 요리 영감 얻음
- **오마이쿡 사용 시나리오**: 
  냉장고 사진과 영수증을 업로드해 재료 자동 등록 → AI가 10분 내 1인분 레시피 추천 → 대체 재료 제안으로 바로 조리 가능 → 조리 중 궁금한 점은 채팅으로 질문

- **직업**: IT 회사 마케터
- **거주**: 경기도 아파트 (남편, 초등학생 자녀 2명)

- **주방 환경**: 3구 가스레인지, 에어프라이어, 오븐, 믹서기
- **목표**: 아이들 영양 관리, 저녁 식사 준비 시간 단축 (30분 내)
- **페인 포인트**: 
  - 매일 저녁 메뉴 고민 피로
  - 아이들 알레르기 고려해야 함 (계란 알레르기)
  - 남은 재료 활용법 모름
- **행동 패턴**: 
  - 주말 대량 장보기

  - 건강 레시피 검색 자주 함
- **오마이쿡 사용 시나리오**: 
  프로필에 아이 알레르기 정보 입력 → 주간 식단 플래너로 일주일 메뉴 자동 생성 → 필요한 재료 리스트 확인 후 주말 장보기 → 매일 저녁 해당 날짜 레시피 보며 조리 → 영양 대시보드로 아이들 섭취 영양소 확인

  - 혼자 먹기엔 레시피 양이 많음 (4인분 기준)
  - 다이어트 중이라 칼로리 체크 필요

- **오마이쿡 사용 시나리오**: 
  1인분 레시피 필터 설정 → 칼로리 낮은 고단백 레시피 추천받음 → 조리 중 "데치기"가 뭔지 AI에게 질문 → 완성 후 영양 정보 확인 및 기록

## 7. 사용자 시나리오

### 시나리오 1: 퇴근 후 빠른 저녁 준비 (민지의 경우)
민지는 밤 9시에 집에 도착했고 배가 고프다. 오마이쿡 앱을 열자 홈 화면에 "오늘의 추천" 3개가 뜬다. 첫 번째 레시피는 "10분 김치볶음밥 (1인분, ★☆☆ 쉬움, 320kcal)"이다. 재료를 확인하니 김치, 밥, 계란, 참기름 모두 보유 중이다. 레시피를 클릭해 단계별로 따라 하며, "중불이 어느 정도인가요?"라고 AI에게 물어본다. AI는 "가스레인지 다이얼 기준 중간 정도, 약 5-6단계입니다"라고 답한다. 12분 만에 맛있는 볶음밥을 완성하고 "완료" 버튼을 누른 뒤 별 5개를 준다.

---

### 시나리오 2: 주간 식단 계획 (수연의 경우)
일요일 오전, 수연은 오마이쿡의 "주간 식단 플래너"를 연다. AI가 월요일부터 금요일까지 저녁 메뉴 5개를 자동으로 제안한다. 수연은 목요일 "닭갈비"를 "된장찌개"로 변경한다. 플래너는 일주일치 필요한 재료 총 목록을 생성하고, 이미 보유한 재료는 자동으로 체크 해제한다. 수연은 장보기 리스트를 보고 마트에서 부족한 재료만 구입한다. 평일 저녁마다 해당 날짜 레시피를 열어 조리하며, 아이들이 잘 먹는 모습을 보고 만족한다.

---

### 시나리오 3: 냉장고 파먹기 챌린지 (지훈의 경우)
지훈은 이사 전 냉장고를 비우려고 한다. 보유 재료 목록에는 파프리카, 양파, 돼지고기, 두부가 남아 있다. "AI 추천" 버튼을 누르자 "두부김치 (1인분, ★★☆ 보통, 280kcal, 매운맛 🌶️🌶️)"가 추천된다. 레시피를 보니 고춧가루가 부족한데, 대체 재료로 "청양고추 + 간장"이 제안된다. 지훈은 청양고추를 사용해 조리를 시작한다. 중간에 "두부는 언제 넣나요?"라고 물으니 AI가 "돼지고기가 70% 익었을 때 두부를 넣으세요. 두부가 부서지지 않게 조심히 뒤집어주세요"라고 답한다. 완성 후 사진을 찍어 인스타그램에 공유한다.

---

### 시나리오 4: 영수증 인식으로 간편 등록 (민지의 경우)
민지는 마트에서 장을 보고 집에 돌아왔다. 영수증 사진을 오마이쿡에 업로드하자 AI가 자동으로 "양파 1개, 당근 2개, 돼지고기 300g, 우유 1L, 계란 10개"를 인식해 재료 목록에 추가한다. 민지는 추가 확인 후 "저장" 버튼을 누르고, 바로 "오늘의 추천"에서 새로 산 재료로 만들 수 있는 "돼지고기 야채볶음"을 발견한다.

## 8. 기술 스택 권장사항

### Frontend
- **Framework**: React (TypeScript)
  - 컴포넌트 기반 설계로 유지보수성 확보
  - 타입 안정성으로 버그 감소
- **Styling**: Tailwind CSS
  - 빠른 UI 프로토타이핑
  - 일관된 디자인 시스템 구축
- **State Management**: 
  - React Context API (간단한 전역 상태)
  - TanStack Query (서버 상태 관리 및 캐싱)
- **Routing**: React Router
- **UI Library**: Radix UI 또는 shadcn/ui (접근성 우수한 컴포넌트)
- **Form Handling**: React Hook Form (성능 최적화)
- **Image Handling**: 
  - Unsplash API (레시피 대표 이미지)
  - 사용자 업로드 이미지 저장소 연동

### Backend
- **Framework**: Node.js + Express 또는 Next.js API Routes
  - JavaScript/TypeScript 풀스택 개발 통합
  - Next.js 사용 시 SSR/SSG 활용 가능
- **Database**: 
  - PostgreSQL (관계형 데이터 저장: 사용자, 재료, 레시피 기록)
  - Supabase (인증 + DB + Storage 통합 솔루션)
- **Authentication**: 
  - Supabase Auth (이메일, 소셜 로그인)
  - JWT 토큰 기반 세션 관리
- **File Storage**: 
  - Supabase Storage 또는 AWS S3 (영수증, 사진 업로드)

### AI/ML
- **LLM API**: 
  - **OpenAI GPT-4 Turbo**: 대화형 어시스턴트, 레시피 생성
  - **Google Gemini Pro**: 비용 효율적인 대안
  - 프롬프트 엔지니어링으로 재료 기반 레시피 생성 최적화
- **Image Recognition (OCR)**: 
  - **Google Cloud Vision API**: 영수증 텍스트 추출
  - **OpenAI Vision API**: 냉장고 사진 재료 인식
- **Response Caching**: 
  - 동일한 재료 조합 요청 시 캐싱으로 API 비용 절감

### 배포 및 인프라
- **Hosting**: 
  - Vercel 또는 Netlify (프론트엔드 자동 배포)
  - Supabase (백엔드 서비스)
- **CI/CD**: GitHub Actions (자동 테스트 및 배포)
- **Monitoring**: 
  - Sentry (에러 트래킹)
  - Google Analytics (사용자 행동 분석)

### 개발 도구
- **Version Control**: Git + GitHub
- **API Testing**: Postman 또는 Insomnia
- **Design Tool**: Figma (UI/UX 디자인)
- **Project Management**: Notion 또는 Linear

### 보안 및 개인정보 보호
- **API Key 관리**: 환경 변수 (`.env` 파일, Vercel Secrets)
- **데이터 암호화**: HTTPS 통신, 비밀번호 해싱 (bcrypt)
- **Rate Limiting**: API 남용 방지 (예: LLM 요청 제한)
- **개인정보 보호**: 
  - GDPR/PIPA 준수
  - 사용자 데이터 최소 수집
  - 탈퇴 시 데이터 완전 삭제

## 12. 결론

**오마이쿡**은 단순한 레시피 검색 도구를 넘어, **AI가 사용자의 냉장고와 상황을 이해하고 최적의 요리를 제안하는 개인화 서비스**입니다. 재료 낭비를 줄이고, 의사결정 피로를 해소하며, 누구나 쉽게 건강한 식사를 준비할 수 있도록 돕습니다.

핵심은 **사용자 경험의 단순함**과 **AI의 똑똑함**입니다. 영수증 한 장으로 재료를 등록하고, 클릭 몇 번으로 오늘 저녁 메뉴가 결정되며, 조리 중 궁금한 점은 AI와 대화하며 해결합니다.

MVP 출시 후 사용자 피드백을 적극 반영하여, 한국의 요리 문화에 최적화된 서비스로 발전시킬 것입니다. 궁극적으로는 "오늘 뭐 먹지?"라는 고민이 사라지는 세상을 만드는 것이 목표입니다.
